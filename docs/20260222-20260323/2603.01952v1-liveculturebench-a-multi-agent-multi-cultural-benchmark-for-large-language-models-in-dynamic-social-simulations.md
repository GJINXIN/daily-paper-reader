## 摘要
大语言模型（LLMs）正越来越多地被部署为自主智能体，但目前的评估主要集中在任务成功率上，而非文化适当性或评估者的可靠性。我们推出了 LiveCultureBench，这是一个多文化、动态的基准测试，它将 LLM 作为智能体嵌入到模拟城镇中，并根据任务完成情况和对社会文化规范的遵循程度对其进行评估。该模拟将一个小城市建模为位置图，其中的合成居民具有多样化的人口统计学和文化背景。每个片段为一名居民分配一个日常目标，而其他居民则提供社会背景。基于 LLM 的验证器生成关于规范违反和任务进度的结构化判断，我们将其汇总为捕捉任务与规范权衡以及验证器不确定性的指标。通过在不同模型和文化背景下使用 LiveCultureBench，我们研究了：(i) LLM 智能体的跨文化鲁棒性；(ii) 它们如何在有效性与规范敏感性之间取得平衡；以及 (iii) LLM 作为评判者（LLM-as-a-judge）的评估在何时对于自动化基准测试是可靠的，以及何时需要人工监督。

## Abstract
Large language models (LLMs) are increasingly deployed as autonomous agents, yet evaluations focus primarily on task success rather than cultural appropriateness or evaluator reliability. We introduce LiveCultureBench, a multi-cultural, dynamic benchmark that embeds LLMs as agents in a simulated town and evaluates them on both task completion and adherence to socio-cultural norms. The simulation models a small city as a location graph with synthetic residents having diverse demographic and cultural profiles. Each episode assigns one resident a daily goal while others provide social context. An LLM-based verifier generates structured judgments on norm violations and task progress, which we aggregate into metrics capturing task-norm trade-offs and verifier uncertainty. Using LiveCultureBench across models and cultural profiles, we study (i) cross-cultural robustness of LLM agents, (ii) how they balance effectiveness against norm sensitivity, and (iii) when LLM-as-a-judge evaluation is reliable for automated benchmarking versus when human oversight is needed.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）被广泛应用于自主智能体，现有的评估体系存在两个主要缺陷：
*   **缺乏动态文化评估**：现有的文化评估（如 CDEval）多基于静态的问答（QA），无法反映智能体在追求长期目标时，如何处理任务效率与社会文化规范（Socio-cultural Norms）之间的冲突。
*   **评估器可靠性不明**：虽然“LLM 作为裁判”（LLM-as-a-judge）被广泛用于社交模拟，但其在处理微妙文化行为时的可靠性和透明度尚未得到充分研究。
**LiveCultureBench** 旨在通过一个动态的、多智能体模拟城镇环境，评估 LLM 智能体在完成日常任务的同时，遵循特定社会文化规范的能力。

#### 2. 方法论：核心思想与关键技术
该框架采用模块化设计，模拟一个包含 1000 名具有真实人口统计特征居民的小镇：
*   **环境建模**：城镇被建模为一个无向图 $G=(V, E)$，每个节点代表特定场所（如医院、办公室、餐厅）。每个场所关联特定的**文化规范**（源自 CultureBank 数据集）和**可用动作**。
*   **智能体设计**：
    *   **目标智能体（Target Agent）**：受试模型，被分配一个包含多个子任务的全天计划。
    *   **支撑智能体（Supporting Agents）**：由 LLM 驱动，被赋予“社交压力”任务，即在互动中诱导或挑战目标智能体去违反文化规范，以测试其鲁棒性。
*   **验证系统（Verifier Agent）**：
    *   **多维指标**：在每个时间步评估任务进度、规范违反、角色忠实度、上下文意识和对话连贯性。
    *   **符合预测（Conformal Prediction）**：引入符合语言建模（CLM）技术，通过校准规则生成包含多个候选判断的集合，从而量化验证器的不确定性并控制评估风险。

#### 3. 实验设计
*   **数据集与场景**：
    *   **人口分布**：基于澳大利亚墨尔本的人口普查数据（年龄、性别、职业、国籍等）生成 1000 个智能体画像。
    *   **文化背景**：涵盖英国、中国、越南、德国、希腊等 11 种主要文化。
*   **对比方法（受试模型）**：
    *   **商业模型**：Gemini 2.5 Pro, Gemini 2.5 Flash。
    *   **开源模型**：Qwen 3 (8B/14B/32B), Llama 3 (8B/70B), Ministral 3 Reasoning (8B/14B)。
*   **Benchmark 设定**：模拟从早上 7:00 到晚上 22:00 的一天，每个模型运行 1000 次独立的模拟片段。

*   **推理框架**：开源模型使用 **vLLM** 进行推理，Gemini 系列通过官方 API 调用。

#### 5. 实验数量与充分性
*   **实验规模**：每个 LLM 后端均进行了 1000 次模拟，总计产生了数万个交互步。
*   **维度覆盖**：实验涵盖了跨文化鲁棒性分析、多文化干扰分析、不同场所（办公室 vs 公园）的规范遵循分析，以及验证器的可靠性校准。
*   **客观性**：通过引入人类标注的 400 个样本进行验证器校准，并使用符合预测技术来量化 AI 裁判的偏见，实验设计较为客观、公平。

#### 6. 主要结论与发现
*   **文化偏见的一致性**：同一家族的模型（如 Qwen 系列）表现出相似的文化偏见模式，通常对西方或其预训练数据占优的文化（如英国、德国、中国）有更好的规范遵循度。
*   **任务与规范的权衡**：LLM 智能体倾向于为了完成任务而牺牲文化适当性，尤其是在任务效率与规范冲突时。
*   **多元文化的挑战**：随着环境中文化多样性的增加，所有模型的规范遵循率（Norm Adherence）均显著下降，Llama 3-8B 的下降最为明显。
*   **场所差异**：在私人空间（如公寓）规范遵循度较高，而在复杂的公共空间（如办公室、商场）违规率显著上升。

#### 7. 优点（亮点）
*   **动态性**：突破了静态 QA 的局限，在长程任务流中观察文化行为。
*   **社交压力机制**：通过支撑智能体主动“诱导”违规，更真实地模拟了现实社交中的诱惑和压力。
*   **风险感知评估**：首次在社交模拟基准中应用符合预测（Conformal Prediction）来量化 LLM 裁判的不可靠性。

#### 8. 不足与局限
*   **文化覆盖有限**：受限于 CultureBank 等现有数据集，无法覆盖所有族裔，且对“文化”的定义存在简化风险（将其等同于国籍或固定规则）。
*   **评估器依赖**：尽管引入了不确定性量化，但核心评估仍依赖 LLM，可能无法完全捕捉极其微妙的人类社交直觉。
*   **计算成本**：大规模的多智能体模拟对 API 调用成本或本地算力有较高要求，限制了其在小规模研究团队中的普及。
