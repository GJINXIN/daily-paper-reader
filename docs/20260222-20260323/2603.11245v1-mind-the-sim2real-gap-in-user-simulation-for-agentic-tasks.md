## 摘要
随着自然语言处理（NLP）评估从静态基准转向多轮交互设置，基于大语言模型（LLM）的模拟器已广泛用作用户代理，承担着生成用户轮次和提供评估信号的双重角色。然而，这些模拟通常被假设为能够忠实反映真实的人类行为，且往往缺乏严格的验证。我们形式化了用户模拟中的 Sim2Real 差距，并首次开展了在真实人类（451 名参与者，165 个任务）上运行完整 τ-bench 协议的研究。我们引入了用户模拟指数（USI）这一指标，用于量化 LLM 模拟器与真实用户交互行为及反馈的相似程度，并据此对涵盖闭源、开源及专门系列的 31 个 LLM 模拟器进行了基准测试。研究发现，在行为上，LLM 模拟器表现出过度合作、风格单一，且缺乏真实的挫败感或歧义，从而创造了一种“简单模式”，导致智能体的成功率虚高，甚至超过了人类基准。在评估方面，真实人类在八个质量维度上给出了细致的判断，而模拟用户则产生了一致更积极的反馈；基于规则的奖励机制无法捕捉到人类用户生成的丰富反馈信号。总体而言，更高的通用模型能力并不一定能带来更忠实的用户模拟。这些发现强调了在智能体开发周期中使用基于 LLM 的用户模拟器时进行人工验证的重要性，并为改进用户模拟模型提供了动力。

## Abstract
As NLP evaluation shifts from static benchmarks to multi-turn interactive settings, LLM-based simulators have become widely used as user proxies, serving two roles: generating user turns and providing evaluation signals. Yet, these simulations are frequently assumed to be faithful to real human behaviors, often without rigorous verification. We formalize the Sim2Real gap in user simulation and present the first study running the full $τ$-bench protocol with real humans (451 participants, 165 tasks), benchmarking 31 LLM simulators across proprietary, open-source, and specialized families using the User-Sim Index (USI), a metric we introduce to quantify how well LLM simulators resemble real user interactive behaviors and feedback. Behaviorally, LLM simulators are excessively cooperative, stylistically uniform, and lack realistic frustration or ambiguity, creating an "easy mode" that inflates agent success rates above the human baseline. In evaluations, real humans provide nuanced judgments across eight quality dimensions while simulated users produce uniformly more positive feedback; rule-based rewards are failing to capture rich feedback signals generated by human users. Overall, higher general model capability does not necessarily yield more faithful user simulation. These findings highlight the importance of human validation when using LLM-based user simulators in the agent development cycle and motivate improved models for user simulation.

*   **研究动机**：如果模拟器表现得不像真人（例如过于合作或信息提供过于完美），那么在模拟环境中表现优异的智能体在现实世界中可能会失败。论文旨在量化这种差距，并评估现有 LLM 模拟器作为用户代理和评估者的有效性。

### 2. 方法论
*   **Sim2Real 差距分类法**：论文提出了一个多维度的分类框架：
    *   **行为差距 (Behavioral Gap)**：包括沟通风格（礼貌、冗长）、信息模式（是否提前提供所有关键信息）、澄清行为（如何表达不确定性）和错误反应（面对智能体错误时的情绪和策略）。
    *   **评估差距 (Evaluative Gap)**：包括结果校准（模拟成功率与真人成功率的差异）和评估对齐（模拟评分与真人评分的一致性）。
*   **关键指标：User-Sim Index (USI)**：
    *   这是一个 0-100 的综合得分，聚合了上述行为维度（使用 **Sørensen–Dice 系数**衡量分布相似度）、**预期校准误差 (ECE)** 和**平均绝对误差 (MAE)**。
*   **流程**：通过真人实验获取“现实”基准，然后让 31 种 LLM 模拟器在相同任务下与智能体交互，计算两者在行为特征和评估反馈上的统计差异。

### 3. 实验设计
*   **数据集与场景**：使用了 **$\tau$-bench** 基准，涵盖“航空”和“零售”两个真实世界的客户服务领域，涉及复杂的数据库操作和政策约束。
*   **Benchmark 设置**：
    *   **真人基准**：招募了 451 名真人参与者，完成 165 个任务，生成交互轨迹并填写包含 8 个维度的质量调查问卷。
    *   **对比方法**：测试了 **31 种 LLM 模拟器**，分为三类：
        1.  **闭源模型**：GPT 系列（含 GPT-5.1）、Claude 系列、Gemini 系列。
        2.  **开源模型**：DeepSeek-V3.1、Llama-4-Maverick、Qwen3 等。
        3.  **专用模拟模型**：CoSER-8B、UserLM-8B 等专门为模拟人类行为微调的模型。

### 5. 实验数量与充分性
*   **实验规模**：对比了 31 个模型，涵盖了目前市面上几乎所有主流的闭源和开源模型，实验组数非常充沛。
*   **充分性与客观性**：

    *   不仅关注任务成功率，还深入分析了 18 个细粒度的行为指标（如礼貌程度、信息前置率等），实验分析非常细致且客观。

### 6. 主要结论与发现
*   **“简单模式”陷阱**：LLM 模拟器普遍表现出**过度合作**。它们会提前提供所有信息（Front-loading），极少表现出真实的挫败感或歧义，这导致智能体在模拟环境中的成功率被虚高（比真人高出约 14%）。
*   **能力不等于保真度**：通用能力最强的模型（如 Chatbot Arena 高分模型）并不一定是最好的用户模拟器。例如，Gemini 系列在通用任务上表现优异，但在用户模拟的 USI 指标上表现参差不齐。
*   **评估失真**：LLM 作为评估者时，对交互质量（如拟人感、流畅度）的评分系统性偏高，而对任务成功的判断则过于保守。
*   **规则奖励的局限**：$\tau$-bench 等基准采用的基于数据库状态的二元奖励（Binary Reward）与人类感知的交互质量几乎**正交（无关）**，无法捕捉用户体验的细微差别。

### 7. 优点
*   **形式化定义**：首次将机器人学中的 Sim2Real 概念引入 LLM 用户模拟领域，并给出了严谨的数学定义。
*   **金标准建立**：通过大规模真人实验纠正了社区对 LLM 模拟器“天然可靠”的盲目假设。
*   **多维度评估**：USI 指标综合了行为和评估两个层面，比单一的成功率指标更具参考价值。

### 8. 不足与局限
*   **领域覆盖**：实验仅限于 $\tau$-bench 的两个客服领域，在软件工程、医疗诊断或社交互动等其他智能体场景下的表现尚待验证。
*   **Agent 固定**：实验中主要固定了智能体模型（GPT-5.2），虽然附录中测试了更换智能体后的排名稳定性，但不同智能体架构对用户行为的诱导作用可能存在更复杂的交互影响。
*   **提示词敏感性**：模拟器的表现可能高度依赖于 Prompt 工程，论文虽使用了标准 Prompt，但未深入探讨 Prompt 优化是否能显著缩小差距。
